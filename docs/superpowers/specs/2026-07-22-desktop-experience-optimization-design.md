# WTH 桌面端体验与功能完善设计

**日期：** 2026-07-22  
**项目：** Wide Thought Host（WTH）桌面端  
**状态：** 设计已确认  
**参考：** `esengine/deepseek-reasonix` 的状态栏、模型切换、右键菜单和抽屉交互

## 目标

在保留现有 Tauri 2、React/Vite、Rust IPC、PTY 终端和设置中心基础上，完成聊天反馈、终端停靠、会话管理、工作区模式、GitHub 登录、模型配置、区域化右键菜单、状态栏和 Windows 品牌图标的整合改造。

本轮采用渐进式整合方案：先统一前端状态和类型化 IPC，再按模块替换现有占位或失效行为，不重写整套界面。最终交付重新生成的 Windows MSI 和 NSIS 安装包。

## 设计决策

### 界面布局与聊天反馈

- 主内容始终保留聊天区，终端改为右侧停靠面板，不再替换整个对话区。
- 终端默认宽度约 `420px`，最小 `320px`，最大不超过可用主区宽度的 `60%`；支持拖动、关闭并持久化宽度。
- 窗口宽度不足 `1000px` 时，终端降级为右侧覆盖抽屉，避免聊天区被压缩到不可用。
- 底部采用全窗口状态栏，横跨左侧导航、聊天和终端，布局与用户提供的截图一致。
- 用户发送消息后立即创建助手占位状态，并在气泡内显示动画和“模型正在思考…”；收到首个文本增量后切换为正文。
- Agent 状态细分为空闲、思考中、工具运行、等待权限、完成和错误，聊天反馈与底部状态栏使用同一状态源。
- 输入区采用稳定的垂直居中布局，发送按钮保持正圆并固定居中，不随单行、多行输入高度偏移。

### 工作区、会话与右键菜单

- 工作区选择器新增“不在工作区中工作”。进入该模式后隐藏文件树、禁用工作区文件工具，聊天仍可使用，终端默认进入用户主目录。
- `workspace_root` 从强制路径调整为可选值；文件树、Git、Agent 工具和终端分别处理无工作区状态，不使用上一次目录冒充当前工作区。
- 会话删除成功后通过原子状态操作同时移除会话、消息缓存和流式状态；若删除的是当前会话，则选择相邻会话，无剩余会话时回到空状态。
- 会话删除默认要求确认；后端失败时恢复原状态并显示明确原因。
- 使用 React portal 实现统一的自定义右键菜单基础组件，自动处理视口边界、焦点、键盘关闭和主题样式。
- 会话菜单提供打开、重命名、置顶/取消置顶、导出和删除；置顶顺序和名称持久化。
- 工作区菜单提供更换工作区、无工作区模式、在资源管理器打开、复制路径和在终端打开。
- 文件与目录菜单提供打开、重命名、复制路径、在资源管理器显示、在终端打开和删除。
- 聊天消息菜单提供复制、复制 Markdown、重新生成和从此处分支；输入框保留剪切、复制、粘贴和全选等原生文本操作。
- 终端标签菜单提供新建、重命名、复制当前目录、清空和关闭。

### 状态栏

- 状态栏使用可配置项目集合，支持显示/隐藏和排序；默认显示模型、工作区、Git 分支、Agent 状态、会话 Token、上下文占用和费用。
- 模型来自当前 Provider 和模型设置；工作区显示路径简称或“无工作区”；Git 分支由 Rust IPC 在当前工作区读取。
- 本轮及会话 Token 来自 Agent usage 事件并累加；上下文占用按累计 Token 与模型上下文窗口计算。
- 命中率、费用、余额仅在 Provider 返回数据或存在明确价格配置时显示真实值；无法获得时显示 `—`，不估算、不伪造。
- 状态栏数据按会话隔离，切换会话时恢复该会话最近状态；无活动会话时显示全局配置和空闲状态。

### 模型与 API 设置

- 模型设置改为“供应商列表、供应商详情、模型列表”结构，不再只能新增和删除供应商。
- 支持新增、编辑、启停、删除、测试连接、设为默认供应商、更新或清除 API Key。
- 支持 OpenAI 兼容、Anthropic、DeepSeek 等常用预设及自定义协议；敏感字段继续写入 Windows Credential Manager。
- 支持从供应商动态获取模型、过滤非聊天模型、搜索、刷新、设置默认模型及覆盖上下文窗口、视觉能力和推理能力。
- 新会话创建时读取当前 Provider、模型与能力配置快照；正在运行的会话不热替换配置。
- 连接测试和模型拉取使用脱敏错误信息，区分网络失败、认证失败、协议不兼容和响应格式错误。

### GitHub 登录

- 保持 GitHub OAuth App Device Flow，不使用 Client Secret，继续使用公开 Client ID `Ov23lixsafitocX2Xgvj`。
- 令牌轮询请求补充 `grant_type=urn:ietf:params:oauth:grant-type:device_code`，这是当前错误的直接代码缺陷。
- 严格使用 GitHub 返回的 `interval`；收到 `slow_down` 后增加轮询间隔，并处理等待授权、拒绝、过期、取消和限速。
- 用户界面显示中文状态与错误，保留验证码复制、打开浏览器、取消登录和退出登录。
- Token 保存到 Windows Credential Manager，用户公开资料可缓存用于离线展示；未登录不限制本地功能。
- 外部人工前置条件：GitHub OAuth App 必须启用 **Enable Device Flow**，不需要配置或提供 Client Secret。

### Windows 图标与安装包

- 从用户 Logo 生成透明方形母版和完整 Windows 图标尺寸，统一 `32x32`、`128x128`、`256x256`、高分辨率 PNG 与多尺寸 ICO。
- Tauri 窗口、任务栏、托盘、release EXE、MSI/NSIS、开始菜单和桌面快捷方式均引用同一套品牌资源。
- 清理重复托盘创建入口，避免资源覆盖；检查并解决 Windows 资源合并警告对 EXE 图标的影响。
- 安装验收同时检查安装包资源和安装后快捷方式目标；如二进制图标正确但系统仍显示旧图标，提供 Windows 图标缓存刷新步骤。

## 接口与状态调整

- `WorkspaceInfo` 明确支持无工作区状态，Rust 应用状态中的工作区根目录改为可选路径。
- 前端会话 store 新增原子操作：删除会话、重命名、置顶、恢复删除和选择相邻会话。
- `AgentEvent` 增加统一的状态与 usage 数据，至少覆盖思考、工具运行、等待权限、文本增量、usage、完成和错误。
- 新增 `StatusBarState` 与 `StatusBarItemConfig`，统一承载模型、工作区、Git、运行状态、Token、上下文、费用和可见性顺序。
- Provider DTO 增加启用状态、默认模型、模型能力覆盖和模型刷新结果；API Key 仍只通过凭据 IPC 写入，不回传明文。
- 新增 Git 状态 IPC、资源管理器打开 IPC，以及会话重命名、置顶和导出 IPC；所有路径操作必须限制在用户明确选择的目录范围内。
- GitHub Auth 状态机补全轮询间隔、错误码、取消和过期信息，前端只消费判别联合状态。

## 数据流与失败处理

1. 发送消息时，前端先写入用户消息和思考占位，再调用 Agent IPC；Agent 事件驱动聊天内容与状态栏同步更新。
2. 工作区切换写入桌面设置并广播统一状态；文件树、终端和新 Agent 会话从该状态读取，已有 Agent 会话保持快照稳定。
3. 会话菜单先更新本地列表，再调用持久化 IPC；失败时使用保留快照回滚。
4. 终端面板仅控制显示和尺寸，PTY 生命周期由 Rust 管理；进程退出或目录失败只关闭对应标签，不影响聊天内容。
5. Provider 编辑时普通配置原子写入版本化设置，API Key 单独写入凭据存储；任一环节失败都不覆盖原有效配置。
6. 状态栏订阅统一状态，不自行重复请求数据；缺少数据时显示 `—`。
